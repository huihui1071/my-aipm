# OneBox OS

一个把任务路由和个人长期记忆合在一起的智能工作台：

- 前台是 `AI Task Router`
- 后台是 `Personal Memory OS`

用户只输入一句话，系统先判断这更像 `Clarify / Search / Write / Analyze / Act` 中的哪一种任务，再按需调用长期记忆，并把路由理由和引用来源展示出来。

当前默认导入了一组个人长期记忆，已经按类整理成可检索的档案卡片，覆盖：

- 基础档案与求职状态
- 时代天使 / 得物 / 蔚来的核心项目资产
- 模型选型、RAG、Prompt、评估闭环等方法论
- 面试答题骨架、表达提醒与补充经历

每条记忆现在采用双层结构：

- `摘要卡`：高密度结论，优先用于检索和路由
- `详细卡`：补充背景、动作、结果、追问点，用于生成更完整的回答

## 产品定位

OneBox OS 想解决的不是“信息收不进来”，而是“信息进来了以后，什么时候该被重新调出来”。

它把两层能力合在一起：

- 前台是 `AI Task Router`
- 后台是 `Personal Memory OS`

用户只需要输入一句话，系统先判断当前更适合澄清、搜索、写作、分析还是执行，再按需调用长期记忆，并把路由理由和引用来源展示出来。

## 当前范围

### 已做

- 一个输入框，支持自然语言输入
- Router 五种模式：
  - `Clarify`
  - `Search`
  - `Write`
  - `Analyze`
  - `Act`
- 长期记忆库：
  - 手动导入笔记、截图摘录、语音转写、网页剪藏
  - 支持固定、忽略、删除
- 结果页可解释：
  - 显示路由结果
  - 显示路由原因
  - 显示召回到的记忆来源
- 轻量执行：
  - 保存 AI 输出到记忆库
  - 生成下一步待办
- 真实产品模式：
  - 支持配置 `OpenAI-compatible` Base URL / Model / API Key
  - 若配置完成，查询会优先调用真实模型生成
  - 若配置失败，会自动回退到本地启发式模式
- 数据可迁移：
  - 支持导出记忆为 JSON
  - 支持导入 JSON 记忆包

### 暂不覆盖

- 自动同步微信、Notion、飞书
- 真正跨平台外部执行 Agent
- 复杂权限体系
- 多用户协作
- 真 OCR / 真 ASR

## 如何体验

直接在浏览器打开 [index.html](./index.html)。

如果想把它真正用起来，而不只是看原型，建议先做 3 步：

1. 在右侧 `接入真实模型` 面板填入 Base URL、Model 和 API Key
2. 点击 `测试连接`
3. 用真实问题去跑一次，例如自我介绍、项目讲稿、面试准备

推荐用这 3 个问题体验：

1. `基于我的个人记忆，帮我总结适合对外讲的 3 个核心卖点。`
2. `我在时代天使项目里的评估方法论是什么？`
3. `如果我明天要面 AI 产品经理，最该优先复习什么？`

## 设计原则

- 只在必要时澄清，不把每个问题都打断成问答树
- 只调用必要记忆，不让历史内容淹没当前任务
- 所有关键结果都尽量给出来源，降低“答得像对但其实无据可查”的风险
- 先把任务判断和知识调用做对，再逐步扩展执行能力

## 部署方式

当前版本是纯前端静态应用，可以直接部署到 GitHub Pages。

推荐目录结构：

```text
deploy/
└── onebox-os/
    ├── index.html
    ├── styles.css
    └── app.js
```

这样部署后，访问路径会是：

`https://<your-github-pages-domain>/onebox-os/`
